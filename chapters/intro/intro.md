
In the traditional Internet architecture, a transport connection is identified by a pair of Internet protocol (IP) addresses and port numbers. This identification model assumes that endpoint addresses remain stable throughout the lifetime of the connection. However, in many contemporary network environments, address stability is not guaranteed. When a host changes its point of attachment to the network and acquires a new IP address, the transport layer can no longer associate subsequent packets with the existing connection state. As a result, even though the communicating application remains active, the session is terminated.

Address change becomes problematic in two related but distinct situations. In mobility, a host typically operates with one IP address at a time, but the address may change during an ongoing communication session. In multihoming, by contrast, a host may maintain multiple IP addresses simultaneously, for example, through different network interfaces. The set of usable addresses may also change over time, so the two phenomena may overlap in practice.

Mobility and multihoming expose a broader architectural problem, where session identity and network location are tightly coupled. This coupling arises from underlying principles of the Internet architecture. Since communication state is maintained at the endpoints, ongoing exchanges are identified through address information carried in packets.

At a basic level, Internet communication extends across multiple packet transmissions. To sustain communication across these transmissions, communication state must be maintained to describe the packets that have been sent, received, and acknowledged. According to the end-to-end principle [@saltzer1984end], this state is implemented at the endpoints to support reliable communication, while the network provides only best-effort delivery. Since the network does not track active sessions, endpoints must determine the association between arriving packets and stored communication state. This association is typically determined using address information, so the endpoint identity is effectively tied to its network location.

However, early Internet specifications did not define the relationship between identity and location as an explicit design principle. Instead, the relationship was implied and assumed from the implementation of transport protocols. In the transmission control protocol (TCP), for example, connection state is associated with the combination of source and destination IP addresses and port numbers. Provided that the values remain stable, packets can be mapped to the correct session state. Once the values change, later packets are treated as belonging to a different connection. The mechanism is simple and efficient, but is dependent on a stable network attachment.

Contemporary deployment environments often lack the required degree of stability: hosts may be multihomed; address translation devices may rewrite source addresses and ports; routing conditions may change independently of application behavior. Under these conditions, the correspondence between the communicating entity and its current network location becomes fragile. As a result, communication can fail even when the application endpoints remain active.

The tension between communication continuity and address dependence has been recognized for decades. In response to this issue, researchers have proposed architectural responses at different layers of the stack. These responses include network-layer indirection mechanisms, such as mobile IP, identity-location separation approaches, such as the host identity protocol (HIP), transport-layer extensions, such as the stream control transmission protocol (SCTP), and multipath TCP (MPTCP), and application-layer techniques that work around transport limitations. Despite considerable technical merit, the proposed approaches have faced significant deployment and compatibility constraints in the open Internet.

Given the deployment and compatibility constraints faced by these earlier efforts, QUIC provides a practically deployed transport-layer design that directly addresses the dependence of connection continuity on stable IP address and port pairs. Standardized by the Internet Engineering Task Force (IETF) in RFC 9000 [@rfc9000], QUIC introduces explicit connection identifiers carried in packet headers. Since these identifiers are independent of network-layer addressing, packets can continue to be associated with an existing connection even when the host IP address changes. This separation between connection identity and the current address pair makes QUIC a useful case for evaluating how far transport-layer mechanisms can mitigate the identity-location problem in practice.

Therefore, the purpose of this thesis is to determine the extent to which QUIC connection migration mitigates the identity-location problem at the transport layer by conducting a literature-based analysis complemented by a small-scale exploratory evaluation using an existing QUIC implementation. The evaluation compares QUIC connection migration with a TCP best-case baseline in a controlled test environment. The remainder of the thesis is divided into five sections. Section 2 presents the architectural background and defines the identity-location problem in greater detail. Section 3 provides an overview of QUIC and HTTP/3. Section 4 analyzes connection migration mechanisms and related extensions. Section 5 presents the experimental setup and results. Section 6 synthesizes the findings, discusses the limitations of QUIC connection migration, and identifies directions for future work.
